# PWM 控制器芯片需求文档

## 1. 项目背景

本项目以一个极简的 PWM（脉宽调制）控制器为载体，完整演示数字芯片从 RTL 设计到 GDS 输出的全流程。核心目标是**学习和验证开源 EDA 工具链**，而非实现复杂的电路功能。

### 1.1 项目目标

- 使用最简单的设计，跑通 RTL → GDS 完整流程
- 掌握 Sky130 (130nm) PDK + OpenLane2 工具链
- 建立可复用的数字芯片开发流程模板

### 1.2 选择 PWM 的理由

| 对比项 | PWM | UART TX | FemtoRV32 |
|--------|-----|---------|-----------|
| 代码行数 | ~25 行 | ~50 行 | ~1500 行 |
| 状态机 | 无 | 有（4 状态）| 有（多层）|
| 验证难度 | ⭐ | ⭐⭐ | ⭐⭐⭐⭐ |
| 全流程时间 | 4–5 天 | 2–3 周 | 2–3 月 |

PWM 只有计数器 + 比较器，逻辑一眼看穿，适合把全部精力放在学习工具链上。

## 2. 功能需求

### 2.1 基本功能

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| FR-01 | 8 位 PWM 输出 | 必须 | 支持 256 级占空比调节（0%–100%） |
| FR-02 | 占空比可配置 | 必须 | 通过 8 位 `duty` 输入端口实时设置 |
| FR-03 | 同步复位 | 必须 | 高有效同步复位，复位后计数器清零、输出低电平 |
| FR-04 | 连续输出 | 必须 | 计数器自动溢出回绕，持续产生 PWM 波形 |

### 2.2 功能描述

- **PWM 波形生成**：内部 8 位计数器在每个时钟上升沿递增 1，溢出后自动回绕到 0。当计数器值小于 `duty` 时输出高电平，否则输出低电平。
- **占空比控制**：`duty` 值范围 0–255，对应占空比 0%–100%（实际分辨率为 1/256 ≈ 0.39%）。
- **复位行为**：`rst` 高有效期间，计数器强制为 0，`pwm_out` 输出低电平。

## 3. 性能需求

| 编号 | 指标 | 目标值 | 说明 |
|------|------|--------|------|
| PR-01 | 工作频率 | ≥ 50 MHz | 时钟周期 ≤ 20 ns |
| PR-02 | PWM 频率 | 工作频率 / 256 | 50 MHz 时 PWM 频率 ≈ 195 kHz |
| PR-03 | 占空比分辨率 | 8 位（256 级） | 最小步进 ≈ 0.39% |

---

## 4. 接口需求

| 编号 | 端口 | 方向 | 位宽 | 说明 |
|------|------|------|------|------|
| IR-01 | `clk` | input | 1 | 系统时钟 |
| IR-02 | `rst` | input | 1 | 同步复位，高有效 |
| IR-03 | `duty` | input | 8 | 占空比设置，0–255 |
| IR-04 | `pwm_out` | output | 1 | PWM 输出信号 |

## 6. 验收标准

| 编号 | 检查项 | 通过标准 |
|------|--------|----------|
| AC-01 | RTL 代码 | 不超过 30 行，端口与需求一致 |
| AC-02 | 仿真验证 | 所有测试用例通过，波形占空比与 duty 值一致 |
| AC-03 | 综合结果 | 无时序违例，标准单元数 20–50 个 |
| AC-04 | 布局布线 | OpenLane2 流程无 ERROR 结束 |
| AC-05 | DRC 检查 | violation = 0 |
| AC-06 | LVS 检查 | net/device match |
| AC-07 | GDS 输出 | KLayout 可正常打开版图 |
